# 前端 UI/UX 专业优化实施方案

本方案旨在通过代码重构解决视觉层次、响应式适配、动效性能、交互体验及无障碍访问等六大核心问题。

## 1. 设计系统与配置升级 (Tailwind Config)
**目标**：建立符合 WCAG 标准的色彩系统与现代化的响应式断点。
- [ ] **更新 `tailwind.config.js`**：
    - 新增 `fold: '880px'` 断点（适配折叠屏）。
    - 新增 `tablet: '744px'` 断点。
    - 覆写 `colors.text.muted` 为 `#64748b` (Slate 500)，确保对比度 > 4.5:1。
    - 定义 `touch-target` 工具类，确保最小点击尺寸为 44px。

## 2. 动效性能重构 (CSS Animation)
**目标**：消除布局抖动 (Layout Thrashing)，确保 60fps 流畅度。
- [ ] **重构 `frontend/src/assets/animations.css`**：
    - **骨架屏/流光效果**：移除 `background-position` 动画。改用伪元素 (`::after`) 配合 `transform: translateX()` 实现 GPU 加速动画。
    - **进度条动画**：同样改为 `transform` 实现。
    - **阴影优化**：将 `box-shadow` 过渡改为伪元素 `opacity` 过渡（如果可行），或减少阴影扩散半径的动画幅度。

## 3. 组件交互体验升级 (Vue Component)
**目标**：优化移动端触摸体验与键盘访问体验。
- [ ] **升级 `frontend/src/components/ui/BaseButton.vue`**：
    - 添加移动端适配逻辑：在小屏幕下强制 `min-height: 44px` 和 `min-width: 44px`。
    - 引入 `:focus-visible` 状态，替代原有的 `:focus`，并添加 `ring-2 ring-offset-2` 的高亮动效 (transition-all duration-200)。

## 4. 无障碍与语义化增强 (Home.vue)
**目标**：提升屏幕阅读器兼容性。
- [ ] **优化 `frontend/src/views/Home.vue`**：
    - 为所有装饰性图标（Lucide icons）添加 `aria-hidden="true"` 属性。
    - 在课程列表区域添加 `aria-live="polite"` 属性，确保筛选和加载更多时的内容变更能被朗读。
    - 检查颜色对比度，确保状态文本具有足够的辨识度。

## 5. 关键渲染路径优化 (HTML)
**目标**：降低 LCP (最大内容绘制) 时间。
- [ ] **优化 `frontend/index.html`**：
    - 移除 CSS 中的 `@import` 字体加载方式。
    - 在 `<head>` 中添加 `<link rel="preconnect">` 和 `<link rel="stylesheet">` 标签，直接加载 Google Fonts，并保留 `display=swap`。

## 6. 验证环节
- [ ] 验证所有动画属性是否已替换为 `transform` / `opacity`。
- [ ] 验证 Tab 键导航时的焦点光圈动画。
- [ ] 验证移动端按钮的可点击区域尺寸。
